OSS Etiquette v1.0 KERNAMON

Follow these guidelines to be able to engage and contribute to OSS Communities.

Guidelines

  1. Read the Documentation First: Always and periodically check the README.md and CONTRIBUTING.md files.
  2. Respect the Project's Scope and Vision: Propose changes aligned with the project goals and strategy.
  3. Be Clear and Concise in Communication: Use simple, professional language anf technical jargon, if necessary. Avoid using any slang unknown to others.
  4. File Issues Responsibly: Include all relevant details for efficient problem resolution.
  5. Make Thoughtful Pull Requests: Validate and test your code to ensure that your code is ready and aligns with the project's style and objectives.
  6. Be Patient and Grateful: Respect the maintainersâ€™ time and efforts.
  7. Give Back to the Community: Help others with reviews, feedback, translations and documentation.
  8. Avoid Making Demands: Suggest changes politely without self-entitlement or arrogance.
  9. Follow Licensing Rules: Adhere to license terms like AGPLv3 or Apache License 2.0 (among many others) and give proper attribution.
  10. Respect Time Zones and Cultures: Be mindful of global contributors and different cultures.
  11. Engage in Discussions: Provide constructive feedback and avoid dismissive behavior.

Conclusion

By following these guidelines, you can contribute positively to OSS Communities, build relationships and increase the chances of your contributions being well-received by others.
